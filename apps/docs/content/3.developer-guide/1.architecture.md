# Architecture & Technology Stack

VaultTracker is a modern Nuxt 3 application designed for performance and offline-first capabilities.

## Core Technologies

- **Framework**: [Nuxt 4](https://nuxt.com)
- **UI Library**: [Nuxt UI](https://ui.nuxt.com) (Tailwind CSS based)
- **State Management**: [Pinia](https://pinia.vuejs.org)
- **Database**:
  - **Client-side**: IndexedDB (via `idb`) for offline storage
  - **Server-side**: Redis (via `unstorage`) for synchronization
  - **Local Development**: Filesystem driver for `unstorage`

## Key Concepts

### Offline-First Sync
The application uses an event-sourced synchronization mechanism. Changes are applied optimistically to the local Pinia state and persisted to IndexedDB. Background workers sync these changes with the server (Redis) when online.

### Storage Abstraction
We use `unstorage` to abstract the server-side storage, allowing seamless switching between Redis (production) and Filesystem (development).

::code-group
```typescript [nuxt.config.ts]
storage: {
  sync: process.env.REDIS_URL
    ? { driver: 'redis', url: process.env.REDIS_URL }
    : { driver: 'fs', base: '.data/sync' }
}
```
::
